解决Centos7系统下vsftpd上传文件不报错但ftp目录下没有文件

​ 请注意：一定要注意你系统环境是Centos7还是Centos6，这个很关键。因为这两种系统下vsftpd的安装和配置过程都有很大差别。

三、文件上传不报错但目录下没有上传的文件

​ 解决思路：这个问题的原因还是vsftpd服务器环境没有搭建成功，原因有四个方面：
​ 1、检查是否是被动模式
​ 打开/etc/vsftpd/vsftpd.conf文件，查看对应的配置是否设置正确，如不正确进行改正；这个主要是因为防火墙并没有开放所有端口，也是出于安全考虑，所以我们需要检查这个配置文件，让vsftpd指定数据传输端口。
​ 2、检查防火墙规则
​

​ 查看防火墙开放了哪些端口，vsftpd需要的端口保证已经全部开放。
​ 打开防火墙的ftp服务。
​

​ 3、文件夹权限问题
​ 查看ftp文件夹的用户权限，保证目录下的文件和文件夹的所有者和用户组都是ftpuser，修改权限参照上述。
​ 4、系统版本造成的问题
​ 如果是CentOS7系统，在设置虚拟用户的时候需要手动添加数据文件.db，并配置PAM认证方式，就是本文上述的PAM认证方式。但如果是Centos6系统，我们就不需要添加这些文件，并保留原有的/etc/pam.d/vsftpd文件不变，从上述的三个方面查找原因。

利用hexo搭建个人博客

发表于 2019-04-25

搭建个人博客

​ 网上关于搭建个人技术博客的方案主要有以下几种：

  • 1、基于PHP的WordPress，这种方案需要自己注册域名和PHP主机；

  • 2、静态博客。这种方案是依赖各种开源代码托管平台（当然还有其他的网站），利用jekyll、hexo等博客生成器在托管平台上搭建博客；

  • 3、VPS搭建。如果买了VPS，可以随意搭建动态或者静态博客；

  • 4、利用现有的博客服务（这里就不举例了，请自行网上搜索）。（摘自知乎）

​ 我这里选择采用hexo来搭建自己的博客，主要为了记录自己在学习过程中遇到的问题。利用hexo搭建博客很简单，晚上教程也有很多，这里不做过多赘述。简单列一下流程。

搭建博客流程

​ 由于hexo是基于Node的静态博客，npm工具是必不可少的。在搭建之前，需要配置一下环境。这里是利用github平台搭建博客，当然还可以通过码云等网站平台构建。附上我的博客让大家看下效果。

环境搭建

​ 这个过程只需要配置三种环境：

​ 1、安装git bash：下载地址，在桌面右键会多出两个git相关的选项。即表示安装成功。

​ 2、安装Node：下载地址，桌面右键点git bash here，输入node -v，显示版本号即成功。

​ 3、安装Hexo：先在本地合适的位置创建blog文件夹（将来写的博文就在这里），然后在这个文件夹里右键，点git bash here，输入npm install hexo-cli -g命令安装，键入hexo -v，显示版本号即安装成功，最后键入hexo init完成初始化。hexo中文官网。（这里hexo更新了安装命令，当初在这坑里呆了一会）

关联github上的仓库

​ 如果之前安装过git，你可能也已经完成SSH绑定（github账户和本地主机的一种关联方式），那么你只需在github上新建一个远程仓库，命名格式为：你的github账号名字.github.io。然后直接到下一步。通过git config –global user.name和git config –global user.email查看自己本地的用户名和邮箱名。如果没有配置，在原来命令的基础上加上“名称”即可。

​ 如果，你还没有一个github账号，或者未关联过远程仓库，甚至没用过git，请移步廖雪峰的git教程。

​ 执行完上述命令后，在git控制台中可以看到server正在运行，这个时候打开浏览器输入http://localhost:4000，即可看到自己的搭建的博客。

​ 先别高兴，你看到的只不过是本地的电脑上的博客，并不能远程访问。要想远程访问，需要输入输入不同的命令。

写博客

  • 新建博客：hexo new 文章名，然后在blog文件夹中的sources里_post中就可以看到。
  • 内容支持Markdown语法，在生成的文件中，用Typora打开，最上面的会有题目、日期信息和标签，如果需要给当前博文添加标签，同样遵守冒号后面加一空格，否则不予识别的原则。
  • 刚提交到github上博客可能不会立即响应，等待5分钟即可。

数据的Json序列化

java中的序列化和反序列化，是为了让数据在网络上的传输和前台的解析。正常的字符或者对象是无法在网络上传输，需要转换成xml或者json格式的二进制流数据。

json的介绍

​ JSON(JavaScript Object Notation)是一种轻量级的数据交换格式，JSON采用完全独立于语言的文本格式，使json成为理想的数据交换语言。

基础结构

​ json建构于两种结构：

  • 1、“名称与值”对的集合； 例如{“firstname”，”sun”}
  • 2、值的有序列表。在不同语言中，它被理解为数组。

{“people”:[
{“firstname”,”sun”},
{“firstname”,”yang”}
{“firstname”,”zhang”}
]}

Jackson需要添加的maven依赖

​ 忽略NULL字段和类中不需要json序列化的属性
